# 创建审计任务第 5 步“确认提交”红框复刻任务书

## 目标

在现有 Vue 3 Demo 上重构 `/tasks/create` 的第 5 步“确认提交”，以用户提供的 `1603 × 1008` 原始 PNG 为视觉事实源。页面在提交前集中展示九类能力配置、全局输出设置、右侧确认摘要和底部操作，并在常见桌面窗口下保持协调比例、无页面级横向溢出、无无意义大面积空白。

## 当前边界与假设

- 本轮只修改确认提交阶段及其直接依赖的纯逻辑、定向测试和视觉验收材料。
- 保留现有全局顶栏、左侧导航、路由和第 1–3 步业务行为；只在确有布局依赖时做最小衔接。
- 参考图红框是功能分区标注，不属于最终 UI。
- 参考图文件实际桌面基准为 `1603 × 1008`；实现不能依赖固定视口高度。
- 不新增第三方依赖，不升级 Vue/Vite，不改无关页面，不清理用户已有改动。
- 每次代码改动后只运行与改动直接相关的测试、构建或视口检查，不跑全量测试。

## 可验证成功标准

- 确认提交页可由第 4 步解析/模板设置阶段进入，步骤条当前项为 5，第 4 步显示完成态，且“上一步”“保存草稿”“提交任务”行为明确。
- 九个能力配置块按参考图内容、层级和相对尺寸完整呈现；报告生成和报告审核占用宽卡布局。
- 全局输出与过程设置包含 AI 生成标识、多模型复核、过程版本、导出留痕四项。
- 右栏完整呈现任务概要、九项已选能力、五项资料及状态、九项预计输出、风险提示。
- 设备形态只验收两类：显示器 `1603×1008 / 1600×900 / 1920×1080`，笔记本电脑 `1440×900 / 1366×768`。
- 显示器采用原始比例并随可用区域适度放大；笔记本保持完整双栏和全部信息层级，使用整页等比缩放，不改成抽屉、单列或压缩字段。
- 两类设备均无页面级横向溢出、卡片重叠或大面积空白；手机和平板不作为复刻目标。
- 新增纯逻辑先写失败测试再实现；视觉改变用同视口截图和 DOM 尺寸检查验收。
- 最终对照参考图至少执行两轮：首轮列差异并修复，末轮重新检查所有红框区域。

## 多 Agent 分工与提示词

### Agent A：参考图规格

> 只读分析参考图，按红框量化页面结构、尺寸、间距、字体、颜色、边框、控件和响应式不变量；不修改文件。输出像素边界、内容层级、比例关系和十大失真风险。

### Agent B：现状代码审计

> 只读审计现有 Vue 页面、领域模型、样式、测试和启动方式；定位第 4 步入口与可复用部分；不修改文件。输出文件/行号、现状差距、保留/重构边界和最小验证命令。

### Agent C：响应式与验收

> 只读制定第 4 步的桌面断点、主栏/右栏收缩规则、空白控制和多视口自动截图方案；结合现有技术栈，不修改文件。输出视口清单、CSS 策略、命令、风险和回退策略。

## 防遗忘规则

1. 每次开始实现、接收代理结果、修改代码或进入下一阶段前，重新阅读本任务书的“目标”“成功标准”“红框区域清单”和“当前阶段”。
2. 调查事实追加到 `notes.md`；不依赖聊天上下文记忆尺寸或结论。
3. 未经定向测试和浏览器截图证据，不勾选 P7–P9 或任何对应红框。
4. 每个代码增量只完成一个逻辑目标，并记录改动文件、相关测试、截图视口和未解决差异。
5. 代理报告只作为输入；主线程必须复查源码、差异和验证输出后才能采用。
