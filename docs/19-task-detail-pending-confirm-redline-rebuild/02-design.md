# 任务详情“生成结果·待确认”设计规格

## 设计目标

将 `/tasks/detail?taskId=TASK-20250428001&tab=results` 构建为九类能力生成完成后的人工确认工作台。参考图 1589×997 是唯一视觉基准；红框仅定义分区，不进入页面。

## 页面架构

```text
AppLayout（任务详情专属壳变体）
├─ 左导航 164–196px
├─ 顶栏 56px
└─ TaskDetail
   ├─ 中央工作区 minmax(0,1fr)
   │  ├─ TaskDetailHeader
   │  ├─ CapabilityResultGrid
   │  ├─ PendingConfirmationTable
   │  └─ VersionExportPreview
   └─ EvidenceTracePanel 276–328px
```

任务详情使用专属 class 和 scoped 样式，不修改通用 `.panel`、`.table-wrap`、`.drawer` 的既有行为。

## 数据与状态

- `createTaskResultState()` 返回新的页面初始状态：9 个能力、15 个待确认事项、V1.0 草稿、选中制度比对首项。
- `getPendingCount(state)`、`getAbilityStatus(state, abilityId)` 和分页均由事项数据派生。
- `confirmItem`、`excludeItem`、`saveItemNote`、`confirmAbility` 返回不可变的新状态；重复操作幂等。
- `canSubmitReview` 只有在待确认数为 0 且版本已保存时为真。
- 页面交互只保存在当前页面内存；刷新恢复 mock 初始态，不声称持久化。

## 视觉布局

- 1589×997：左栏约 195px、顶栏 56px、中央外槽约 1059px、追溯栏 328px。
- 任务摘要三层约 83/75/41px。
- 九能力区 3×3，卡高约 108px，列间 12px、行间 10px。
- 待确认表显示 5 条紧凑行，表体独立滚动；分页保留 15 条、10 条/页的业务口径。
- 底部版本与文件预览紧跟表格，不使用自动撑底。
- 页面底色 `#f5f7fa`，卡片白色，边框 `#e5eaf0`，品牌红 `#b40000`，链接蓝 `#1677ff`。
- 标题/分区/正文/表格字号分别约 20/14/11–12/9–10px。

## 响应式

- 三列为 `clamp(164px,12.4vw,196px) minmax(0,1fr) clamp(276px,20.7vw,328px)`。
- 使用 CSS 密度变量缩放间距、行高和控件尺寸；不使用单独 `transform:scale()`。
- 1440×900 和 1600×900 密度约 0.90；1366×768 密度约 0.84。
- 1366px 仍保留三列能力卡；中央与追溯栏各自纵向滚动。
- 小于 1280px 才折叠左栏、将追溯栏改为覆盖式。
- 1920px 宽时中央区弹性增宽，不添加大面积左右留白。

## 交互

- 卡片与表格“查看依据”打开准确事项；卡片入口优先选该能力首条待确认事项。
- 确认、排除、编辑说明同步更新顶部计数、行状态、卡片状态和日志。
- 表格说明上限 200 字；说明为选填。
- 提交复核按钮保持可点击，有未处理项时阻断并定位表格；归档任务当前禁用。
- 第九能力支持模拟上传与执行检查。
- 右栏五标签：来源条款、关联文件、智能体会话、版本快照、操作日志；内容跟随事项。
- 右栏支持关闭和 Esc；重新打开默认回到来源条款。

## 可访问性与失败反馈

- 所有动作使用原生 button、a、input、select、textarea。
- 图标按钮提供 aria-label；状态同时有文字，不只靠颜色。
- 阻断、保存、导出、上传和处理结果通过页面 toast/notice 提供可观察反馈。
- 没有真实文件或后端时使用明确的“演示预览/已记录”反馈，不伪装为真实成功。
