# 任务中心红框复刻任务书

## 纠偏结论

本次用户指向的正确目标是 `/tasks`（任务中心），不是 `/tasks/create`（创建审计任务）。截图红框的任务统计、筛选、Tab 表格、右侧提醒与快捷入口都属于任务中心。`/tasks/create` 保持已有创建向导，不在本轮继续改动。

## 截图区域与实现映射

1. **右上创建入口**：固定红色“创建审计任务”，跳转 `/tasks/create`。
2. **顶部五个状态卡**：全部任务、执行中、待确认、已归档、失败/异常；数值、较昨日变化和图标同排，点击后筛选表格。
3. **筛选栏**：审计单位、任务类型、审计期间、状态、关键词、查询、重置；标签在上、36px 控件在下。
4. **任务 Tab/表格/分页**：Tab 与状态卡联动；紧凑表格显示任务编号、名称、类型、单位、期间、阶段、进度、风险、创建人、更新时间、状态、操作；底部显示总数和分页。
5. **右侧信息栏**：今日待确认事项、失败任务、最近操作日志、快捷入口，纵向四块，禁止使用泛化“状态说明”替代。
6. **统一演示数据开关**：全局唯一 `store.demoDataMode` 是有数据/无数据的单一真相；全局顶栏提供“演示数据 有 / 无”开关，并与工作台、文件中心一致。无数据时所有统计为 0、表格为清晰空态；有数据时展示完整截图结构。

## 第二轮截图对齐记录

本轮重新以参考图逐区复核，避免将红框标注误实现为产品边框。

1. **页面骨架**：改为左主列（创建、五卡、筛选、Tab/表格/分页）与右栏（306px）从顶部同时开始；不再让五卡和筛选跨越右栏。
2. **顶部与状态卡**：`/tasks` 顶栏只保留“任务中心”；创建按钮固定在左主列右缘。五卡均为中性白底，状态选择由 Tab 红色下划线表达。
3. **表格与分页**：补齐每行“查看详情 + 当前动作”，10 条/页、上一页、1/2/3、下一页和跳页输入；详情页读取 `taskId`/`tab`，显示对应任务名称与处理页签。
4. **右栏链路**：待确认、失败任务的“查看全部”直接切换相应 Tab；每条事项可进入带任务上下文的详情页。
