# 资料解析与预检红框复刻任务书

## 目标

以用户提供的红框效果图为唯一视觉真值，重构“资料上传后的解析与预检”页面。页面必须完整表达解析进度、目录树、文件解析清单、字段映射与预检结果、失败解释、处理流程追踪和底部处置动作，并在不同窗口宽度下保持比例协调、信息密度合理、无页面级横向溢出或大面积无效空白。

## 当前约束

- 先审计、拆解和确认设计，再修改业务代码。
- 现有项目是 Vue 3 + Vite；优先保留可复用的全局布局、路由、Font Awesome 图标和 store 行为。
- 截图中的红框是区域标注，不是产品 UI，不得实现为红色边框。
- 只实现截图和设计说明要求的功能，不扩展后台、鉴权或持久化能力。
- 每次改动后只运行与改动直接相关的测试；最终再运行构建与浏览器验收。
- 现有工作区已有大量用户改动，禁止清理、重置或改写无关文件。

## 成功标准

1. 参考图所有红框区域均有一一对应的可见实现，层级、比例、间距和信息密度接近原图。
2. 失败文件可查看详情、重新解析、替换文件；字段不完整文件可进入字段映射；重复/异常文件有可处理动作。
3. 右侧详情能随当前文件变化，并明确失败原因、缺失字段、影响能力、是否阻断提交。
4. 主流程、列表选择、折叠区、分页和底部动作可操作，状态变化一致。
5. 显示器版保持原尺寸完整三栏；笔记本版保持相同完整三栏与信息层级，按 `min(可用宽度 / 1608, 可用高度 / 1014)` 整体等比缩放并水平居中。两档均不得出现页面级横向滚动或大面积无效空白。
6. 同视口截图对比完成，`design-qa.md` 最终写明 `final result: passed`。
