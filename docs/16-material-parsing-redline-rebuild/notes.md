# 资料解析与预检复刻调研记录

## 已确认事实

- 项目技术栈为 Vue 3 + Vite，已安装 Font Awesome。
- 现有 `/audit-report/material/precheck` 页面是通用卡片式“入库前预检”，与参考图的解析工作台结构差距很大。
- 参考图为桌面高密度业务界面，核心不是图表看板，而是步骤条、解析统计、目录树、文件清单、字段映射/预检、详情解释与处理时间线的协同。
- 工作区已有大量用户文件和暂存改动，后续只允许触碰本轮明确列出的文件。

## 视觉量测结果

- 参考图原始尺寸为 1608 × 1014；有效应用区域约 1587 × 993，右侧和底部深灰截图外框不属于产品界面。
- 红框主区域：顶部解析总览约 1000 × 181；文件树约 204 × 496；解析清单约 755 × 419；字段映射/预检约 762 × 299；右侧详情总容器约 358 × 857。
- 中央清单使用约 28px 行高、10–11px 正文字号的紧凑业务表格；操作列约 139px，需保持可见或 sticky。
- 桌面需要四类滚动/固定关系：全局顶栏；目录树正文 + 底部批量操作；清单表头/正文/分页；右侧详情正文 + 底部操作。
- 参考图核心状态为：76% 进度、8 成功、2 解析中、1 失败、1 重复、1 格式异常；异常行选中，右侧展示“解析失败”，下一步禁用。
- 视觉令牌接近：主红 `#C80000`、成功绿 `#078A3F`、信息蓝 `#1677FF`、警告橙 `#FA8C16`、错误红 `#F5222D`、浅灰边框 `#E4E7ED`；红框 `#FA5151` 仅是标注色。
- 响应式基线：≥1440px 保持三栏；1200–1439px 收窄/折叠全局导航并允许右详情收起；768–1199px 详情改抽屉、目录树可收起；<768px 清单改最小列/卡片，步骤条横滚。

## P0 视觉验收

- 同视口骨架与各区域尺寸误差目标为 ±3px，中央清单、右栏和底部按钮不得相互覆盖。
- 必须还原失败态数据、异常行选中、状态语义与禁用下一步；按钮和错误原因不得被截断。
- 文件树批量操作、清单表头/分页、中央流程按钮、右侧操作区必须在各自滚动容器中始终可见。
- 图标必须使用真实图标库；同一状态在顶部、表格、预检汇总和时间线中保持同一图标/颜色语义。

## 现有实现审计结论

- 正确目标不是 `/audit-report/material/precheck`，而是 `/tasks/create` 中“资料选择”与“模板与输出设置”之间缺失的“资料解析与预检”子阶段。证据是参考图面包屑、五步名称与当前创建流程完全一致，且当前“下一步：解析资料”会直接跳到模板摘要。
- 推荐把解析实现为第 4 步下的 `parsing → template` 两个子阶段：进入第 4 步先展示参考图解析工作台，问题处理完成后同一步切换到模板设置；这样既还原截图的第 4 步高亮，又保持业务链路连贯。
- `/audit-report/material/*` 属于另一条“报告资料导入/入库”生命周期，必须保留，不应拿目标页覆盖。
- 现有 `TaskCreate.vue` 可复用五步步骤条、资料上传来源和上游资料状态；现有通用 `DataTable`、`MetricGrid` 和 `.tree` 结构不支持目标密度与联动，不应为了本页修改全局组件。
- 推荐最小边界：`TaskCreate.vue` 负责流程切换；新增 `MaterialParsingWorkspace.vue` 负责页面布局和局部交互；新增 `materialParsingState.js` 负责纯状态转换；新增定向测试。只有第三段面包屑确有需要时才小改 `AppLayout.vue`。
- 本页局部使用现有 Font Awesome，不扩充手写 SVG；不修改全局 `table`、`DataTable`、`store/mock` 或 `/audit-report/material/*` 页面。

## 当前运行基线

- 项目可通过 `npm run dev -- --port 5173 --strictPort` 启动；当前控制台基线为 0 error / 0 warning。
- 旧 `/audit-report/material/precheck` 在 1608/1440/1280px 无页面级横向溢出，但 1024/768/390px 分别溢出约 106/214/528px；根因是通用表格 `min-width: 860px` 与父网格收缩约束不足。
- 旧页面只有“返回元数据/确认入库”，缺失目标图中的所有解析、选择、失败处置、映射、时间线和分页交互，不能作为视觉或功能基线。
- 根 `design-qa.md` 的通过结论属于旧 `/tasks/create` 基础页面，不覆盖本轮解析子阶段。

## 待确认的唯一产品差异
