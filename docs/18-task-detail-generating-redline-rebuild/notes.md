## 现有实现与最小改造边界

- `taskCenterData.js` 已有目标任务、生成中状态和 63% 进度，可以作为任务头上游。
- `mock/index.js` 的能力目录含九类能力，但运行数据仅四项；全局页签只有十项且缺“智能体会话”。
- 不直接扩写全局 `capabilityRuns`、`operationLogs` 或持久化 `db`，避免 localStorage 旧数据和其他页面共享数据污染目标页。
- 建议新增任务详情专属数据模块，并让能力卡、四项统计、右栏当前步骤和执行明细从同一状态模型派生。
- `AppLayout.vue` 只增加 `/tasks/detail` 专属 shell class、面包屑和顶栏控制；不修改通用任务创建判断。
- `TaskDetail.vue` 可整体重写；只在职责清晰时拆为“能力执行区”和“执行日志右栏”两个专属组件。
- 样式必须使用 `task-detail-*` 专属命名，避免全局 `.panel/.section-grid/table` 断点误伤。

- 九卡有 9 项，底表实际可辨识 8 行；“审计规范生成”与“监督共享信息分析”在第 5 行附近存在文字/产出错配。

- 当前默认不悄悄修正这些矛盾；设计确认时明确选择视觉优先或数据一致性优先。

## 暂定设计判断

- 视觉目标是一个约 1380×930 的高密度桌面工作台，不适合把每个区块做成松散的大卡片。
- 推荐“宽度同比缩放 + 低高度令牌压缩 + 超宽自然流”的混合方案，不按高度直接缩到 0.773。
- 1366×768 以宽度缩放约 0.861，同时把未缩放设计高度压缩约 10%，保留 3×3 与右栏同排；最多允许约 40px 纵向滚动，不继续牺牲可读性。
- 1440×900 以宽度缩放约 0.908，只轻微减少纵向间距；1586×993 1:1；1920×1080 不放大整壳，主内容自然增长但限制在约 1600px，右栏最多约 360px。
- 视觉保真桌面下限为 1366px；1200px 以下才允许右栏下移和九能力改为两列。
- 卡片与执行明细应由同一份九能力状态数据派生，防止进度、状态和操作不一致。

## 实施与验收结论

- 新增 `TaskDetailGenerating.vue`、`TaskCapabilityExecutionGrid.vue`、`TaskExecutionLogRail.vue` 和 `taskDetailExecutionData.js`，通过共享快照驱动任务头、九卡、右栏和明细表。
- `TaskDetail.vue` 保留草稿、待确认、归档实现，通过 `resolveTaskDetailView` 分派生成中状态；`AppLayout.vue` 使用同一解析器决定专属壳，避免内容和壳状态分裂。
- 参考任务显式 `state=generating` 时保留设计快照 63%；其他生成中任务按任务中心记录覆盖编号、名称、负责人和总进度，浏览器实测 `TASK-20250425004` 正确显示“固定资产审计 / 78%”。
- 1586×993 对照迭代：先校正浏览器无头外框造成的 24px×92px 伪空白，再修品牌、元数据列位、页签、右栏间距和字号层级；最终截图为 `screenshots/task-detail-generating-1586x993-final.png`。
- 局部放大对照覆盖任务头、流程+九卡、右栏日志+阻断；字体、间距、颜色、图标和业务文案均无剩余 P0/P1/P2。
- 1366×768 与 1440×900 的侧栏高度按 `100vh / scale` 反算，缩放后白色侧栏覆盖整个视口，不再留下灰色空带。
- 内置浏览器完成暂停/继续、页签往返、结果抽屉开关、全部日志聚焦和控制台验证；浏览器 error/warning 为 0。
- 最终只运行任务详情相关测试：57/57 通过；`npm run build` 通过，Vite 171 modules transformed。
