# 审计规范结果页研究记录

## 已确认现状

- 技术栈为 Vue 3 + Vue Router + Vite，无 UI 组件库。
- `/audit-standard/draft` 当前仅有来源、草稿、建议三栏和整稿确认/锁定，不具备目标页结构与逐条交互。
- `/tasks/detail` 的 `handleViewResult` 当前只显示提示，不导航。
- 通用 `.panel` 的大圆角、阴影和松散密度与效果图冲突，目标页需要页面专属样式命名空间。
- 工作树无提交且大量文件已暂存；必须使用手术式补丁，不提交、不清理用户内容。

## 视觉基线

- 原图实测 1586×995。
- 左导航约 202px；顶部约 62px；内容为左条件约 209px、中央约 788px、右追溯约 348px。
- 主红约 `#c70000`，成功绿约 `#059b58`，链接蓝约 `#1677ff`，真实边线为 1px 浅灰。
- 信息密度高：正文 11–13px、控件 30–32px、轻圆角、少阴影。
- 原图进度文案为 186/320（58%），进度条视觉长度偏短；实现统一按 58%。

## 响应式决策

- 用户希望“放大和缩放”，因此采用 1586×995 整体等比缩放作为桌面默认，不采用各模块独立重排。
- 画布顶部对齐并水平居中；根文档无横向滚动。
- 低于 1200px 才允许右侧追溯覆盖或局部滚动等可用性降级。

## 保护边界

- 不扩大 `store/index.js` 巨型可变状态；新交互采用独立纯状态模块。
- `TaskDetail.vue` 只增加 ability 5 的路由跳转，不把新页状态塞回任务详情。
- 不改其他任务详情状态和不相关业务页面。
