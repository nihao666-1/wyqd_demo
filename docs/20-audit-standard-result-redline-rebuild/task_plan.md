# 审计规范生成结果页红框复刻任务书

## 目标

将现有 `/audit-standard/draft` 从稀疏的三栏草稿页重构为任务详情中“审计规范生成”能力的完整结果页，并把 `/tasks/detail` 的该能力“查看结果”接入此页。以用户提供的 1586×995 效果图为唯一视觉基准，覆盖来源依据、人工确认、版本保存、操作留痕和 Excel 导出等演示闭环。

## 唯一视觉基准与原则

- 参考图：`C:\Users\ZHUANZ~1\AppData\Local\Temp\codex-clipboard-a2b50bbb-6fbd-4f1e-8868-93f89a16c968.png`。
- 图片实测尺寸：1586×995；红框只定义功能分区，不属于最终 UI。
- 1586×995 为 1:1 黄金画布；1366×768、1440×900、1600×900、1920×1080 通过整体等比缩放保持比例，画布顶部对齐、水平居中，不允许页面级横向溢出。
- 低于 1200px 才允许必要的可用性降级；表格、详情轨可局部滚动，但不得打散桌面三列骨架。
- 不实现真实后端、真实 AI 或真实文件生成；所有操作必须在前端状态内有可观察反馈和留痕。

## 红框区域拆解

1. 全局壳：202px 左导航、62px 顶栏、任务中心选中、面包屑、通知/帮助/用户区。
2. 标题动作条：审计规范生成；返回任务、重新生成、保存草稿、导出 Excel。
3. 任务元信息：被审计单位、审计期间、审计类型、当前模板、生成状态。
4. 左侧生成条件：模板、内外规制度范围、监管案例范围、风险事项、历史规范版本、输出格式、开始生成、提示说明。
5. 中央进度：六步步骤条、四项来源完整性、生成日志、条目总进度、模型与并行能力。
6. 中央草稿：186 条规范表格、风险/确认状态、查看依据、编辑条目、确认、排除、分页；右侧重点建议。
7. 右侧来源追溯：制度条款、关联文件、人工处理、折叠项、八步流程时间线、确认引用/替换来源/确认引庇、查看原文。
8. 左下导出记录：6 张 Excel/Word 文件卡。
9. 中下版本时间线：V0.1、V0.9、V1.0 三节点。
10. 右下快捷入口：任务详情、版本对比、操作留痕。
