# 实施笔记

## 当前已知

- 现有 `/tasks` 有数据态已完成左主列 + 306px 右栏布局。
- 现有空态只是有数据表格骨架归零，与新参考图差异很大。
- 新图要求空态拥有独立的筛选结构、6 卡、状态说明和推荐演示路径，因此应拆为独立组件，而不是继续堆叠条件样式。
- 全局状态单一真相保持 `store.demoDataMode`。

## 需要复核

- 仓库内是否已有空任务插画或可直接使用的图标资产。
- 1588 × 996 基准视口下右栏和主列表的实际高度。
- 新空态和旧数据态之间的顶部标题口径。

## 实施结论

- 新增 `TaskCenterEmptyState.vue`，由 `TaskList.vue` 根据 `store.demoDataMode` 选择空态或原有数据态。
- 空态单位与负责人分别来自 `store.db.organizations`、`store.db.auditPersonnel`，离岗人员不会显示。
- Font Awesome 只用于新空态的标准图标和收件箱插画组合；未使用 emoji、手绘 SVG 或占位符。
- 桌面测量：筛选 `(224,70,1092,133)`、6卡 `(224,215,1092,120)`、列表 `(224,345,1092,420)`、推荐路径 `(224,777,1092,164)`、状态说明 `(1332,218,228,720)`。
- 全局切换验证：空态6卡/9状态；导入后数据态28条/5卡；切回“无”恢复完整空态。
- 浏览器控制台错误与警告为0。
