# 费用审计分析总览页复刻任务书

## 目标
- 页面：`/expense/anomaly/dashboard`
- 标题与面包屑：费用审计分析
- 目标：按效果图红框拆分，复刻已有费用数据后的费用审计分析总览页，兼顾 1600px、1440px、移动窄屏的比例自适应。

## 红框区域拆解
1. 顶部筛选区：被审计单位、审计期间、费用类别、部门、员工、数据来源、查询、重置。
2. 指标卡区：费用总额、预算使用率、异常金额、异常笔数、高风险异常数；每卡含图标、主值、单位、同比上升。
3. 四图表区：预算执行趋势、费用类别结构、人均费用排行（TOP10）、异常类型分布。
4. 下方异常表：超预算未审批、费用违规报销、疑似不合规报销、异常汇总、整改建议；支持查看详情、确认异常、排除异常、导出 Excel。
5. 右侧详情抽屉：基本信息、凭证信息、审批链路、规则命中、相似记录、人工处理。
6. 全局壳层：左侧费用审计分析子导航高亮，顶部面包屑对齐目标图。

## 多 Agent 分工和提示词
- Agent 1 截图拆解：逐区拆解红框内 UI 结构、文案、尺寸层级、交互入口，输出必须实现项和可近似项。
- Agent 2 代码定位：定位 `/expense/anomaly/dashboard` 路由、布局壳层、旧组件依赖和可能导致回写旧页面的入口。
- Agent 3 验证审计：检查页面是否仍出现旧“费用异常监控”文案，给出构建、局部测试和截图验证清单。
