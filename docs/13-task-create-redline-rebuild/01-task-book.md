# 创建审计任务（基础信息）红框复刻任务书

## 目标与边界

将 `/tasks/create` 重构为“创建审计任务”的第一步：**基础信息**。它必须继承用户参考图的审计后台语言（红色品牌、白色面板、紧凑表单、细线分隔、右侧辅助信息），而不是把任务中心表格照搬进创建页。

本轮只实现并验收第一步页面及其与第二步的前进衔接；五步导航须完整可见，但步骤 2–5 的内容不在本轮重做范围内。不得改变任务中心、工作台、全局路由或其它业务页。

## 用户给定的不可变要求

- 基础字段：任务名称、被审计单位、审计期间、任务类型、负责人、参与人员、任务说明。
- 五步向导顺序：基础信息、选择能力、资料选择、模板与输出设置、确认提交。
- 任务类型应影响后续推荐资料、可选能力和默认模板；本页需明确展示该影响的提示。
- 提供“保存草稿”和“下一步”；点击下一步进入“选择能力”。
- 页面在小窗口和大屏均比例协调、不产生无意义的大面积空白，表单不重叠、不横向溢出。

## 页面分区与细化规格

### A. 全局承载

- 延用现有 `AppLayout` 的侧栏和顶栏，避免重做已存在的导航系统。
- 内容区最大宽度使用 `min(100%, 1440px)`，同时保持 `clamp(12px, 2vw, 28px)` 的左右内边距；超宽屏居中，不拉伸字段至难读。
- 移除旧 `PageHeader` 大块留白式标题区，改为紧凑页头：面包屑、`创建审计任务` 标题和一行辅助说明。

### B. 五步进度条

- 五个步骤始终完整显示，步骤 1 当前态为审计红，后续为中性灰；完成态预留但不在本轮伪造。
- 桌面采用单行五等分；`< 900px` 保持可横向滚动且不压缩中文；`< 560px` 改为带序号的两列/最后一项单列，确保可点击。
- 当前页面初始步骤必须为第 1 步（索引 0），不能误落入“选择能力”。

### C. 基础信息主表单

- 第一行：任务名称（跨两列）、任务类型。
- 第二行：被审计单位、审计期间、负责人。
- 第三行：参与人员（跨两列，可展示已选人员标签）、审计范围/覆盖部门（沿用现有数据，作为补充信息）。
- 第四行：任务说明（全宽，多行）。
- 字段采用真实可编辑原生控件；必填项有 `*` 与表单标签绑定，不靠颜色表达必填。
- 所有控件高度、边框、圆角与参考图筛选控件一致：紧凑、浅灰边线、焦点红色描边。

### D. 类型联动与右侧辅助栏

- 任务类型默认“常规审计”；可选“专项审计”“制度比对”“费用审计”。
- 改变类型时更新右侧的“后续推荐能力 / 建议资料 / 默认模板”三项摘要；这是对用户所述后续影响的可见承诺，下一步可直接消费同一状态。
- 辅助栏还包含“填写提示”和步骤进度说明；每组有图标、标题、1–3 条短文本，禁止堆放装饰性大卡片。

### E. 底部动作与状态

- “保存草稿”保存当前表单状态，并以非阻断的内联成功提示反馈；不跳转。
- “下一步”先校验必填字段；失败时在相应字段旁显示文本错误，并聚焦第一个错误；成功时进入步骤 2。
- “返回任务列表”为次级按钮，不和主操作争夺视觉权重。
