# R1 创建审计任务视觉差距观察记录

## 参考图事实（2026-07-10）

- 新图为 `1603 × 1006` 桌面状态，红框仅表达区域分工。
- 左侧壳层约 `211px`，顶部栏约 `60px`；主栏约 `1055px`，右栏约 `286px`。
- 首步表单不是普通三列卡片：每一行是两组“标签—控件”，控件约 `350 × 38px`；范围与说明为并列文本域。
- 步骤条与基础信息共用一个主面板，数据范围/风险和底部动作各自成面板；右栏有“口径说明”和“预计所需资料”两张卡。

## 当前差距

- 当前步骤条与主表单分离，且页面标题包含与新图不一致的辅助文字。
- 当前主表单为三列字段网格，不能还原参考图的双列标签—控件节奏。
- 缺少数据范围多选、风险等级选择、底部 `当前步骤 1 / 5` 与取消操作。
- 当前右栏以任务类型摘要为主，缺少必填检查和资料必填/可选分级。

## 决策

- 保留 `taskCreateForm.js` 内已有的首步字段、类型联动、人员目录和校验，扩展而不替换其行为。
- 仅在 `TaskCreate.vue` 重排首屏；步骤 2–5 内容不做视觉扩建。
- 视觉验证与行为验证分开记录：前者使用同尺寸截图，后者使用现有轻量 Node 测试与浏览器交互。
