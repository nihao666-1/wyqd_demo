# 审计大模型系统操作演示流程

## 一、文档目标

本文档用于演示“审计大模型系统”的最小功能闭环：从审计工作台查看任务态势，创建审计任务，完成资料和模板配置，提交后进入任务中心观察自动执行进度，再进入分析、报告生成、报告审核和记录中心完成业务闭环。

演示版允许使用前端模拟数据，不要求连接真实后端。演示重点是让观看人员理解系统如何把“任务创建、资料处理、模型分析、人工确认、报告生成、审核复核、操作留痕”串成一条完整流程。

## 二、最小功能闭环定义

本次演示的最小闭环为：

```text
审计工作台
  -> 任务中心
  -> 创建审计任务
  -> 提交任务
  -> 任务中心查看进度
  -> 任务详情处理生成结果
  -> 专项/费用/制度分析结果查看
  -> 报告智能化生成报告
  -> 回传正式报告比对
  -> 报告审核处理问题
  -> 记录中心查看留痕
```

最小闭环必须覆盖以下系统能力：

1. 能看到当前任务、待办、风险提醒和最近生成结果。
2. 能创建并提交一条审计任务。
3. 提交后任务中心新增任务记录，显示任务编号、状态、进度和操作入口。
4. 任务进度可自动模拟后台推进，到 100% 后进入“待确认”。
5. 能进入任务详情查看生成结果并进行人工处理。
6. 能查看专项审计、费用审计、制度与规范相关分析结果。
7. 能生成报告草稿、保存版本、提交复核、导出报告。
8. 能上传正式报告并执行生成稿与正式稿比对。
9. 能进入报告审核处理问题并生成复核记录。
10. 能在记录中心查看关键操作留痕。

## 三、演示前准备

### 1. 打开系统

操作：

1. 打开浏览器。
2. 访问系统地址：`http://192.168.0.115:5178/workbench`
3. 确认左侧导航出现“审计工作台、任务中心、制度与规范、专项审计分析、费用审计分析、报告智能化、文件中心、配置中心、记录中心”。

预期效果：

1. 默认进入【审计工作台】。
2. 页面右上角显示当前用户“审计管理员”。
3. 开发或演示环境中可看到“演示数据 有 / 无”切换。

### 2. 切换演示数据

操作：

1. 点击右上角“演示数据”的【有】。
2. 如需演示首次使用空状态，可先切到【无】，展示空工作台和空任务中心，再切回【有】继续主流程。

预期效果：

1. 【有】状态下，工作台、任务中心、分析页、报告页都有模拟数据。
2. 【无】状态下，系统展示空状态引导入口。

讲解重点：

“演示数据开关用于模拟真实系统中的历史任务和已有数据。正式环境中这些数据来自任务库、文件库、分析结果库和操作日志。”

## 四、主线演示流程

## 1. 进入【审计工作台】查看全局态势

### 1.1 查看工作台总览

操作：

1. 点击左侧导航【审计工作台】。
2. 查看页面中的任务指标、待办事项、风险提醒和最近生成结果。

重点观察：

1. 当前进行中的审计任务数量。
2. 待确认异常、待复核报告、失败任务等提醒。
3. 推荐的业务入口，例如创建任务、处理待办、查看最近结果。
4. 最近任务进度，例如某些任务处于“处理中”“待确认”“失败”等状态。

讲解话术：

“审计工作台是系统入口，审计管理员可以在这里先看全局状态：有哪些任务正在执行、哪些结果需要人工确认、哪些报告需要复核、哪些风险需要优先处理。”

### 1.2 从工作台进入任务办理

操作：

1. 在工作台中找到任务或快捷入口。
2. 可以点击【创建审计任务】或直接从左侧进入【任务中心】。

预期效果：

1. 若点击创建入口，进入【创建审计任务】。
2. 若进入任务中心，先看到任务列表和任务状态分布。

## 2. 进入【任务中心】创建审计任务

### 2.1 查看任务中心

操作：

1. 点击左侧导航【任务中心】。
2. 查看顶部任务状态卡、筛选条件、任务列表和操作列。

重点观察：

1. 任务编号。
2. 任务名称。
3. 任务类型。
4. 被审计单位。
5. 审计期间。
6. 当前阶段。
7. 进度。
8. 风险项。
9. 状态。
10. 操作入口：例如【查看详情】【继续执行】【处理】【查看日志】。

讲解话术：

“任务中心是任务全生命周期入口。创建、执行、待确认、归档和异常任务都会在这里统一跟踪。”

### 2.2 点击创建审计任务

操作：

1. 点击任务中心顶部或筛选区旁的【创建审计任务】。

预期效果：

1. 页面进入【创建审计任务】。
2. 顶部面包屑显示“当前任务：创建审计任务”。
3. 页面显示五步流程：
   - 选择能力
   - 填写基础信息
   - 资料选择
   - 模板与输出设置
   - 确认提交

## 3. 完成任务创建流程

### 3.1 第一步：选择审计能力

操作：

1. 在能力卡片中选择本次任务需要的能力。
2. 推荐演示选择【监管案例舆情分析】。
3. 也可以补充说明其他能力，例如：
   - 制度查询
   - 制度变更
   - 制度比对
   - 审计规范生成
   - 监督共享信息分析
   - 费用审计
   - 报告生成
   - 报告审核
4. 点击【下一步：填写基础信息】。

预期效果：

1. 被选中的能力卡片高亮。
2. 下一步按钮可用。

讲解话术：

“任务创建第一步是选择审计能力。能力决定后续需要哪些资料、执行哪些模型分析、生成哪些结果和报告输出。”

### 3.2 第二步：填写基础信息

操作：

1. 填写或确认任务名称。
2. 选择被审计单位，例如【上海分公司】。
3. 选择审计期间，例如【2025Q1】。
4. 确认任务类型，例如【专项审计】或【常规审计】。
5. 确认负责人和参与人。
6. 填写审计目标、数据范围或业务范围。
7. 点击下一步。

推荐演示字段：

| 字段 | 推荐值 |
|---|---|
| 任务名称 | 上海分公司 Q1 监管案例舆情分析任务 |
| 被审计单位 | 上海分公司 |
| 审计期间 | 2025Q1 |
| 任务类型 | 专项审计 |
| 负责人 | 审计管理员 |
| 业务范围 | 适当性管理、反洗钱、费用报销 |

预期效果：

1. 基础信息保存到任务草稿。
2. 页面进入资料选择阶段。

讲解话术：

“基础信息决定任务归属、权限范围和审计周期。后续资料筛选、结果引用和报告生成都会绑定这些任务上下文。”

### 3.3 第三步：选择或上传资料

操作：

1. 进入【资料选择】。
2. 选择资料来源：
   - 本地上传
   - 文件中心选择
   - 历史任务复用
   - 平台资料复用
3. 推荐演示点击【从平台资料补齐】或选择已有模拟资料。
4. 查看资料完成度、必需资料、可选资料和阻断项。
5. 如果有阻断项，点击【批量上传】或【从平台资料补齐】。
6. 确认资料满足继续条件后，点击下一步。

预期效果：

1. 资料清单展示上传状态和解析状态。
2. 资料完成度提升。
3. 阻断项减少或消失。
4. 可以继续进入模板与输出设置。

讲解话术：

“资料选择阶段会根据所选能力自动生成资料清单。系统会提示哪些资料是必需的，哪些资料可选，哪些资料缺失会阻断后续分析。”

### 3.4 资料解析与预检

操作：

1. 如果页面进入资料解析阶段，查看解析进度。
2. 查看文件解析清单、字段映射、预检结果。
3. 对解析失败、重复文件、格式异常等问题进行处理。
4. 确认资料可用于模型分析后继续。

预期效果：

1. 资料解析状态清晰可见。
2. 可看到已解析、解析中、解析失败等状态。
3. 阻断项处理后进入下一阶段。

讲解话术：

“资料解析是模型执行前的质量门禁。只有资料完整、字段可映射、权限和格式符合要求，系统才会进入后续生成分析。”

### 3.5 第四步：模板与输出设置

操作：

1. 选择任务输出模板。
2. 设置输出内容，例如：
   - 分析明细 Excel
   - 风险建议清单
   - 审计报告草稿
   - 附件包
3. 选择输出格式，例如 Word、Excel、PDF。
4. 确认是否启用来源追溯、多模型复核或导出规则。
5. 点击进入确认提交。

预期效果：

1. 页面展示模板配置摘要。
2. 页面展示输出文件配置。
3. 可以继续提交任务。

讲解话术：

“模板与输出设置决定模型结果怎么沉淀。系统不仅生成分析结果，也会按照报告模板和导出规则形成可审阅、可追溯、可归档的文件。”

### 3.6 第五步：确认提交

操作：

1. 在确认页检查任务名称、审计单位、审计期间、能力、资料和输出设置。
2. 点击【提交任务】。

预期效果：

1. 页面返回【任务中心】。
2. 任务中心新增一条任务记录。
3. 新任务显示任务编号。
4. 初始状态为【生成中】。
5. 初始进度为 `12%`。
6. 操作列显示【查看详情】【继续执行】。

讲解话术：

“提交后任务进入执行队列。演示版使用前端模拟任务记录，真实系统中这里会写入任务表，并由后端调度模型和资料解析任务。”

## 4. 在任务中心查看任务自动推进

### 4.1 观察自动后台进度

操作：

1. 停留在【任务中心】。
2. 不需要手动点击任何按钮。
3. 观察新建任务进度自动变化。

预期变化：

| 时间点 | 进度 | 当前阶段 | 状态 | 操作 |
|---|---:|---|---|---|
| 提交后 | 12% | 生成分析 | 生成中 | 继续执行 |
| 约 3 秒后 | 30% | 资料解析 | 生成中 | 继续执行 |
| 约 6 秒后 | 55% | 能力执行 | 生成中 | 继续执行 |
| 约 9 秒后 | 78% | 风险提取 | 生成中 | 继续执行 |
| 约 12 秒后 | 100% | 人工确认 | 待确认 | 处理 |

讲解话术：

“用户不需要手动改进度。任务提交后由系统后台自动执行。演示版每 3 秒推进一次，用来模拟真实后台调度、资料解析、模型分析和风险提取过程。”

### 4.2 进入生成中详情页

操作：

1. 在任务状态仍为【生成中】时点击【继续执行】。
2. 进入【任务详情】。
3. 默认查看【分析过程】。

重点观察：

1. 顶部任务信息：编号、单位、期间、类型、负责人、创建时间。
2. 总进度。
3. 六步流程：提交任务、文件解析、能力执行、结果确认、报告生成、复核导出。
4. 能力执行进度：制度查询、制度变更、制度比对、监管案例舆情分析、费用审计、报告生成等。
5. 右侧执行日志、当前模型步骤、来源资料快照、失败与重试、待处理提示。

讲解话术：

“继续执行并不是人工推进，而是进入执行看板，查看后台任务正在跑什么、哪些能力已完成、哪些能力还在运行、有没有失败或阻断。”

### 4.3 进入待确认处理

操作：

1. 等待任务中心的新任务进度达到 `100%`。
2. 状态变为【待确认】后点击【处理】。
3. 进入任务详情的待确认工作台。

重点操作：

1. 查看【生成结果】。
2. 查看能力结果卡片。
3. 打开待确认事项。
4. 在右侧依据追溯中查看来源条款、关联文件和证据。
5. 对事项执行：
   - 确认
   - 排除
   - 填写处理说明
6. 处理完成后点击【提交复核】或【导出结果】。

预期效果：

1. 待确认事项数量减少。
2. 操作写入页面日志。
3. 导出结果时显示导出反馈。
4. 提交复核后任务进入后续复核处理链路。

讲解话术：

“模型生成完成后不会直接归档，而是进入人工确认。审计人员可以查看来源依据、确认风险、排除误报，并把处理结果写入留痕。”

## 五、分析结果分支演示

主线任务进入待确认后，可以根据演示重点选择以下一个或多个分支展示。

## 5. 专项审计分析

### 5.1 进入专项审计分析入口

操作：

1. 点击左侧导航【专项审计分析】。
2. 进入【专项审计入口】。
3. 查看监管案例舆情分析、监督共享信息分析等入口。

重点观察：

1. 专项分析指标概览。
2. 监管案例、舆情风险、监管关注点、任务历史。
3. 入口按钮：
   - 新建案例舆情分析
   - 新建共享信息分析
   - 查看结果
   - 查看依据
   - 生成报告
   - 导出

讲解话术：

“专项审计分析用于承接监管案例、舆情、监督共享信息等非结构化或半结构化信息，最终输出监管关注点、风险主题和审计建议。”

### 5.2 查看监管案例舆情分析结果

操作：

1. 点击子菜单【监管案例舆情分析】。
2. 查看监管关注点趋势、风险主题分布、案例来源分布。
3. 在关注点表格中点击【查看详情】。
4. 查看右侧关注点详情。

重点观察：

1. 综合结论。
2. 相关监管案例。
3. 舆情来源。
4. 处罚依据。
5. 相似点分析。
6. 人工处理选项。
7. 导出结果按钮。

讲解话术：

“监管案例舆情分析把处罚案例、舆情信息和业务主题进行聚合，识别出审计应重点关注的问题，并保留案例来源和处罚依据。”

### 5.3 查看监督共享信息分析

操作：

1. 点击【监督共享信息分析】。
2. 查看监督共享资料、标签提取、报告摘要和输出文件。
3. 查看来源文件和引用关系。

重点观察：

1. 共享资料解析结果。
2. 标签与主题聚类。
3. 汇总分析报告。
4. Excel、Word、附件包输出。
5. 来源追溯和引用状态。

讲解话术：

“监督共享信息分析用于把内部监督、检查、整改等资料沉淀为可引用的审计素材，并能继续进入报告生成。”

## 6. 费用审计分析

### 6.1 进入费用审计入口

操作：

1. 点击左侧导航【费用审计分析】。
2. 进入【费用审计入口】。
3. 查看费用数据快照、费用分析任务、异常监控任务和导出报表。

重点观察：

1. 费用数据接入状态。
2. 费用综合分析入口。
3. 费用异常监控入口。
4. 费用趋势分析入口。
5. 历史费用分析任务。

讲解话术：

“费用审计分析聚焦费用数据、审批链路、预算控制和凭证证据，帮助审计人员快速识别异常报销和费用趋势风险。”

### 6.2 查看费用审计总览

操作：

1. 点击【费用审计分析】或【查看费用总览】。
2. 查看筛选条件、指标卡、图表和异常明细。

重点观察：

1. 费用总额。
2. 预算使用率。
3. 异常金额。
4. 异常笔数。
5. 高风险异常数。
6. 费用类别分布。
7. 异常类型占比。
8. 异常明细清单。

讲解话术：

“费用总览帮助审计人员先判断费用结构和异常规模，再下钻到单笔异常和依据文件。”

### 6.3 处理费用异常

操作：

1. 在异常明细中点击【查看详情】。
2. 查看异常详情抽屉。
3. 查看基本信息、凭证信息、审批链路、规则命中和依据。
4. 根据情况点击：
   - 确认异常
   - 排除异常
   - 补充说明

预期效果：

1. 异常状态更新。
2. 处理结果写入记录。
3. 可继续导出异常清单或引用到报告。

讲解话术：

“费用异常不是简单给出结论，系统会展示规则依据、凭证文件和审批链路，让审计人员能判断该异常是否成立。”

### 6.4 查看费用趋势分析

操作：

1. 点击子菜单【费用趋势分析】。

## 7. 制度与规范

### 9.2 上传正式报告并执行比对

操作：

1. 点击上传区域，选择本地 Word、PDF 或模拟文件。
2. 选择比对基准，例如当前生成稿 V1.0。

预期效果：

## 十、演示常见问题

### 1. 任务提交后是否必须人工点击“后台运行”？

不需要。

演示版中，任务提交后会自动启动前端模拟后台进度。用户可以停留在任务中心观察进度，也可以点击“继续执行”进入任务详情查看执行过程。

### 2. 100% 是否代表任务完全结束？

不是。

100% 表示模型生成阶段完成。审计业务上还需要人工确认生成结果，所以任务状态会变为“待确认”，操作入口变为“处理”。

### 3. 报告生成和报告审核是什么关系？

报告生成负责形成报告草稿和来源引用。报告审核负责检查报告格式、内容、依据和风险表述，并形成复核记录。

### 4. 正式报告比对解决什么问题？

它解决“线下修改后的正式报告是否改变了系统生成稿关键结论”的问题。系统会识别内容差异、结构差异、依据差异和风险提示。

### 5. 记录中心为什么放在最后演示？

因为记录中心用于证明闭环完整。前面每一步的关键操作都应该能在记录中心形成留痕。

## 十一、演示收口话术

“这套系统的核心不是单点生成，而是把审计任务从创建、资料准备、模型分析、人工确认、报告生成、报告审核到操作留痕全部串起来。模型负责提升分析和生成效率，审计人员负责确认判断和风险把关，记录中心负责沉淀全过程证据。”

|---|---:|
