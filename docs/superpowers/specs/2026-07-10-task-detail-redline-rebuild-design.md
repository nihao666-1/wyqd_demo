# 任务详情页红框复刻设计

## 1. 背景与目标

现有 `/tasks/detail` 呈现“处理中任务台账”，而目标图呈现“任务刚创建、草稿、尚未上传资料”的统一任务承载页。两者在业务状态、页面拓扑、首屏密度和响应式比例上均不一致，不能通过继续压缩旧卡片解决。

本次采用局部结构重建：保留路由、应用壳、图标组件和基础设计 token，重写任务详情主体，并为任务详情增加受限的应用壳变体。目标是在 1600×1000 基准视口逐区对齐参考图，同时在 1366×768、1440×900、1920×1080 下保持桌面双栏、比例协调、无横向溢出和无大面积无意义空白。

## 2. 唯一视觉与业务基准

- 参考图实际像素约 1593×999，按 1600×1000 设计基准验收。
- 左导航约 212px，顶部栏约 60px。
- 主内容有效宽度约 1353px，工作区按约 75:25 分为左主列和右辅助列，列间距约 9–12px。
- 任务状态固定为草稿，当前版本为 `--`，输入资料为空。
- 仅“创建任务”流程完成，其余步骤均为待开始。
- 参考图上的红框是区域标注，不进入最终 UI。

## 3. 方案选择

采用方案 A：在任务详情范围内重建结构和数据，所有新增样式使用 `task-detail-` 前缀及 scoped CSS；不修改全局 `.panel`、`.btn`、`table` 等裸选择器。

未采用方案：

- 全局组件重构：会影响大量已完成页面，回归面过大。
- 固定 1600×1000 画布整体 `zoom`：基准截图容易接近，但在宽屏产生空带，在窄屏降低可读性，也不利于键盘和浏览器兼容。

响应式采用流体栅格加有限密度档：结构不变，尺寸按断点收紧或放宽。1366–1920 三个桌面档均保持右栏同排。

## 4. 页面架构

```text
TaskDetail.vue
├─ 任务头：返回、标题、草稿标签、元数据、顶部动作
├─ 单行 11 项标签栏
└─ 75:25 详情栅格
   ├─ TaskDetailOverview.vue
   │  ├─ 空资料启动区与三入口
   │  ├─ 三张所选能力卡
   │  └─ 七步任务流程
   └─ TaskDetailRail.vue
      ├─ 下一步提示
      └─ 当前任务操作留痕
```

组件边界只拆两个职责明确的子组件，避免把每张单次使用的卡片继续抽象。`TaskDetail.vue` 负责页面级状态、标签切换和导航；子组件只负责展示本页草稿 fixture。

## 5. 数据契约

新增独立 `draftTaskDetail` fixture，不复用或清空全局任务、文件和日志集合，避免改变任务中心、文件中心和记录中心。

固定内容：

| 字段 | 值 |
| --- | --- |
| 任务编号 | `TASK-20250428001` |
| 名称 | 上海分公司 Q1 常规审计任务 |
| 被审计单位 | 上海分公司 |
| 审计期间 | 2025Q1 |
| 任务类型 | 常规审计 |
| 负责人 | 张三 |
| 状态 | 草稿 |
| 当前版本 | `--` |
| 创建时间 | 2025-04-28 09:15:32 |

资料缺口为财务报表、费用明细台账、相关制度文件，状态均为未上传。所需输出为制度差异分析报告、费用审计分析报告、审计报告（草稿），状态均为待生成。

所选能力严格为制度比对、费用审计、报告生成。操作留痕仅包含一条“审计管理员 / 创建任务 / 成功”。

## 6. 区域设计

### 6.1 应用壳与顶部栏

`AppLayout.vue` 增加独立 `isTaskDetailRoute`，不能把详情页并入当前“创建审计任务”分支。

- 顶部左侧显示“任务中心 / 任务详情”，详情为当前项。
- 详情页隐藏全局演示数据切换，避免参考图不存在的控件挤占空间。
- 通知、帮助、审计管理员保留；通知角标不随空数据模式消失。
- 应用壳增加 `task-detail-shell` 类，任何尺寸覆盖都限定在该前缀下。

### 6.2 任务头

任务头基准高度约 159px：

- 第一行包含返回按钮、24px 任务标题和浅蓝草稿标签。
- 第二行是白色细边框信息卡，包含 7 项元数据和右侧 3 个动作。
- 元数据使用文本列和竖分隔线，不使用只读输入框。
- 按钮顺序固定为继续编辑、上传资料、删除草稿；主按钮深红，后两者白底红边。

### 6.3 标签栏

标签顺序固定为：任务概览、输入资料、分析过程、生成结果、智能体会话、报告与附件、修改记录、复核记录、版本记录、导出记录、操作留痕。

标签保持单行，不显示数量和状态。任务概览使用红字与约 3px 下划线。详情首次进入或 query 非法时强制回到 `overview`，不读取持久化的全局 `activeTaskTab`。

### 6.4 空资料启动区

左主列上半区约 274px：

- 左侧使用本地内联 SVG 文件夹/文档/云上传插画，不依赖外部网络资源。
- 右侧显示“任务已创建，尚未添加资料”和两行说明。
- 三个入口顺序和图标严格匹配参考图。
- 启动区只保留必要留白，不设置超过内容需要的固定 `min-height`。

### 6.5 能力卡

三卡同排、等高，约 126px：

- 制度比对使用绿色图标。
- 费用审计使用橙色图标。
- 报告生成使用蓝色图标。
- 每卡包含标题、待资料标签、说明、输出结果。
- 使用 1px 冷灰边框和 5–6px 圆角，不加重阴影。

### 6.6 下一步提示

右栏约占 25%，包含：

1. 待补充资料（必填）与编辑链接；三行未上传资料。
2. 所需输出（生成后可用）；三行待生成输出。
3. 阻断项；红色警告图标和说明。

组间以 1px 分隔线区分，列表行高控制在约 39–41px。

### 6.7 任务流程

七步横向流程：创建任务、上传资料、解析资料、生成分析、人工确认、保存版本、导出归档。

- 创建任务节点为红色完成态，显示已完成和创建时间。
- 其余节点为灰色待开始态。
- 节点使用等分栅格，连接线由每个节点间的弹性区域生成，不用绝对坐标散布。

### 6.8 操作留痕

右下区与流程区处于同一底部行，顶部对齐误差不超过 12px。使用本页紧凑表格，不复用全局最小宽 860px 的 `DataTable`。

包含全部日志入口、四列表头、一条创建成功记录和底部分页。结果“成功”使用绿色文字。

## 7. 入口行为

- 继续编辑 → `/tasks/create?phase=confirm`。
- 上传资料 → `/tasks/create?phase=materials&source=local`。
- 从文件中心选择 → `/tasks/create?phase=materials&source=fileCenter`。
- 使用模拟数据 → `/tasks/create?phase=materials&source=simulation`。
- 删除草稿 → 打开本页二次确认；确认前不修改任何数据。
- 下一步提示的编辑链接 → `/tasks/create?phase=materials&source=local`。
- 全部日志 → `/records?taskId=TASK-20250428001`。

`TaskCreate.vue` 需以最小改动支持 `phase=materials` 和 `source` 初始化；不得调用全局 `setDemoDataMode('data')`，以免持久化全局状态或插入 notice 推低页面。

## 8. 响应式设计

### 1600×1000

- 作为主叠图验收尺寸。
- 左导航约 212px，顶栏约 60px。
- 任务头、主工作区、流程和日志应在首屏完整出现。
- 区块位置容差 ±4px，主列宽容差 ±8px，字号容差 ±1px。

### 1440×900

- 使用约 0.9 密度档，保留全部桌面结构。
- 标签、元数据、能力卡保持单行。
- 页面允许极少量纵向滚动，但不能产生横向滚动。

### 1366×768

- 使用紧凑密度档，左导航约 176–184px。
- 右栏仍同排，三张能力卡仍同排，标签仍单行。
- 允许合理纵向滚动，不把正文压到物理 12px 以下。
- 关键头部和三个资料入口必须在首屏可操作。

### 1920×1080

- 内容利用新增宽度，右栏控制在约 390–410px。
- 上传区内部内容设最大宽度，避免插画、文案和按钮被过度拉散。
- 1920 以上才限制主内容最大宽约 1680–1720px。

## 9. 可访问性与错误处理

- 所有操作使用 `button`、`RouterLink` 或 `a`，不使用可点击 `div`。
- 图标按钮具备可见或 `aria-label` 名称。
- 标签使用 `role=tablist/tab/tabpanel` 和 `aria-selected`。
- 删除草稿使用语义化 `dialog` 或等价的可聚焦确认区域，支持 Escape/取消。
- 内联插画标为装饰性，不进入无意义的辅助技术朗读。
- 本页不使用 `v-html`。

## 10. 测试与验收

新增 `tests/taskDetailEmptyState.test.js` 和 `test:task-detail` 脚本，只运行本次相关测试。

自动测试覆盖：

1. fixture 字段、资料、输出、能力、流程和日志契约。
2. 11 个标签顺序和默认 overview。
3. 页面及两个子组件可编译。
4. 页面不再导入 PageState、MetricGrid、DataTable。
5. 应用壳存在任务详情独立分支。
6. `phase=materials` 和三种 source 初始化行为。
7. 删除草稿需要二次确认。

视觉验收覆盖 1600×1000、1440×900、1366×768、1920×1080：

- 无横向溢出。
- 左右栏保持同排。
- 标签和能力卡不意外换行。
- 页面末尾无超过 40px 的异常空白。
- 控制台 0 error、0 warning。
- 最终截图再次逐区对照原图，并记录残余差异与修正。

最终只运行：

```powershell
npm run test:task-detail
npm run build
```

不运行无关全量测试。
