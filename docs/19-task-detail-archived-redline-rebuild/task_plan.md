## 并行审计汇总结论

- 当前 `/tasks/detail` 是“办理中通用台账”，实时 `1600×995` 文档高约 `1578px`；不能靠换文案或压间距达标。
- 当前大部分详情数据来自全局 `store.db`，并未按 `taskId` 过滤；版本/导出/复核/留痕只有 `2/2/2/3` 条，与目标 `5/6/1/8` 条不符。
- 通用 `DataTable` 的全局最小宽度和通用 `PageHeader/PageState/MetricGrid` 均不适合本页；归档页应使用专属紧凑组件和 scoped 样式。
- 推荐新增不可变归档快照数据、`TaskDetailArchived.vue`，由 `TaskDetail.vue` 做生命周期分派；`AppLayout.vue` 只加 `/tasks/detail` 专属 shell 分支。
- 响应式基准：逻辑画布 `1600×995`；`scale = clamp(0.85, viewportWidth / 1600, 1)`；1366≈0.854、1440=0.9、1600=1、1920仍为1并居中。
- 右栏任何桌面视口都不得下沉；表格横向溢出只能留在内部容器，不能污染 body。
- 1366×768 启用紧凑高度与内部滚动；右栏中部滚动、底部操作 sticky；不继续缩小正文字号。
- 视觉重点：标题条 53px、元数据 60px、成果卡 84px、表格行 28–32px、右栏 249px；红框 `#FA5151` 是标注层，不属于 UI。
