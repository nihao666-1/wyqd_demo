最后更新：2026-07-13

## 目标

把 `/regulatory/result` 重构为用户最新红框图中的“监管案例舆情分析”执行与结果页。目标是贴近 1586x994 效果图：左侧导航、顶部面包屑、主内容区、右侧关注点详情栏同时成型，并在 1366、1440、1586、1920 宽度下按比例缩放或压缩，避免大面积空白。

## 当前事实

- 当前 `RegulatoryResult.vue` 只是候选池表格和抽屉，和目标图差距很大。
- `src/router/index.js` 导入了 `RegulatoryWorkbench.vue`，但该文件当前缺失，导致 `npm run build` 失败。
- 目标图红框只是功能区域标注，最终页面不能保留红框。
- 本轮只改监管案例舆情分析相关页面、专用 shell、局部测试和任务书。

## 红框区域拆解

### R01 左侧导航与顶部栏

- 左侧栏约 203px，品牌红底白字，高约 57px。
- 当前主项为“专项审计分析”，红色文字、浅红底、左侧红条。
- 子项“监管案例舆情分析”处于当前项，下面有“监管共享信息分析”。
- 顶部栏高约 56px，面包屑为“审计工作台 / 专项审计分析”。
- 右上角通知、帮助、用户区域保持现有全局样式，目标图中通知角标为 12。

### R02 页面标题与筛选区

- 标题“监管案例舆情分析”位于主内容左上，字号约 20px，旁边有信息圆点。
- 筛选区高度约 70px，白底浅边框，六个字段一行排列：被分析单位、指定业务、审计期间、监管机构、风险主题、舆情来源。
- 右侧三个操作按钮：查询、重置、新建案例舆情分析；主按钮红底白字。

### R03 汇总指标区

- 五个横向指标，整体高度约 86px，白底浅边框。
- 指标为监管案例 58、舆情风险 24、监管关注点 126、审计建议 32、已引用到任务 18。
- 每项左侧彩色图标块，中间大数字，底部“较昨日 +N”。
- 分隔线清晰，不能做成过大卡片。

### R04 图表区

- 四个图表一行：趋势、风险主题分布、案例来源分布、高频处罚原因 TOP5。
- 趋势图：红色折线 + 蓝色虚线，近 6 个月坐标。
- 主题分布：环形图中心写“合计 126条”，右侧图例显示百分比。
- 来源分布：蓝色柱状图，显示 21/15/10/8/4。
- TOP5：横向蓝色条，显示 18/16/12/10/8。
- 图表必须用自适应 SVG 或 CSS，不能在缩放时比例失衡。

### R05 结果表

- 标签页：监管案例、舆情风险、监管关注点、审计检查建议、已引用；当前为“监管关注点”。
- 表格展示 5 行：关注点编号、风险主题、关联案例/舆情、业务线、风险等级、相似度、建议检查内容、引用状态、操作。
- 操作包含查看详情、加入审计重点、引用到报告、删除。
- 底部分页显示共 126 条、页码、10条/页、跳至页。

### R06 处理流程

- 一行流程：获取案例、提取风险、生成人工建议、已引用。
- 每步有圆形状态图标、标题、数量、完成时间，中间箭头连接。
- 高度约 76px，密度紧凑。

### R07 历史任务

- 标题“历史分析任务”。
- 3 行历史记录，列为任务编号、任务名称、业务范围、分析期间、状态、关注点数、创建时间、输出结果、操作。
- 操作含查看、复制、删除。

### R08 右侧关注点详情

- 右栏固定约 305px，从顶部栏下方开始到页面底部。
- 标题“关注点详情”，右上关闭。
- 内容块：综合结论、相关监管案例、舆情来源、处罚依据、相似点分析、人工处理。
- 底部固定三个按钮：加入审计重点、引用到当前任务、导出结果。
- 右栏内容要能滚动，底部按钮不能遮挡文本。

## 自适应策略

- 1586x994：以目标图为基准，左侧栏 203px、右侧栏 305px、主内容约 1060px。
- 1366/1440：使用页面级 `zoom` 或 CSS 变量等比缩放到视口内，保持右侧详情栏可见。
- 1920：页面居中或主区适度拉伸，但图表和表格不出现大片空白。
- 1024 以下：右侧详情下移或覆盖显示，主区保持单列/横向表格滚动。
