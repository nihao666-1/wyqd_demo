# 任务中心复刻审计记录

## 当前实现证据

- 路由 `/tasks` 指向 `src/views/tasks/TaskList.vue`；截图的侧栏高亮也为“任务中心”。
- 当前 `TaskList.vue` 由 `PageHeader`、`PageState`、只读 `FilterPanel`、通用 `DataTable` 组成，无法满足截图的紧凑结构。
- 当前 `TaskCreate.vue` 初始 `step = 1`，因此直接落在“选择能力”；基础字段为只读数组，缺任务说明，不符合创建第一步要求。
- 已有 `docs/12-workbench-redline-rebuild/01-task-book.md` 和工作台改动不属于本次路径，必须保留不动。

## 已发现风险

1. 文本说明称“老用户已有任务”，但截图所有统计为 0、表格为空且右栏是状态说明。已按双状态处理。
2. 现有全局 `@media (max-width: 1100px)` 会把通用网格全部压成单列，无法满足任务中心中间断点；页面需有专属断点。
3. 实测旧 `/tasks`：768px 根页面 `scrollWidth=863`；390px `scrollWidth=653`。新页面必须将宽表转换为信息卡。
4. 无 Vitest。对布局采用 Vite + 浏览器截图、控制台与 scrollWidth 证据；功能采用现有 `npm run check` 与 `npm run build`。
