# 任务中心与创建任务向导复刻任务书

## 目标与边界

本轮只重构 `/tasks`（任务中心）和 `/tasks/create`（创建审计任务）。

- `/tasks`：以用户提供的 `1584 × 996` 截图作为**空态像素基准**；实现同骨架的有数据态。
- `/tasks/create`：没有效果图，只按已给出的字段、五步流程与交互说明实现基础信息页；不把它伪称为像素复刻。
- 不修改 `WorkbenchHome.vue` 或既有 `docs/12-workbench-redline-rebuild/`，它们是另一页面的未提交工作。

## 任务中心红框逐区规格（1584 × 996）

### A. 全局壳层与顶栏

- 左侧导航：宽 `212px`，白底，品牌区为 `64px` 高主红；当前“任务中心”浅红底、左侧 `4px` 红色活跃条。
- 顶栏：主区顶部高 `52px`，展示“审计工作台 / 任务中心”；右侧通知红点、帮助、用户和下拉。
- 内容背景为浅灰白；白色仅用于筛选、卡片、表格和右栏，不留无意义的整屏白底。

### B. 标题、筛选与创建入口

- 左区 `任务中心` 标题及筛选总高约 `136px`；标题 `24px`，控件标签 `13px`，控件高 `36px`。
- 字段顺序固定：任务名称、被审计单位、审计期间、任务类型、任务状态、负责人、查询、重置。
- 右区独立放置 `192 × 50px` 的“创建审计任务”主按钮；不挤进筛选网格。
- 查询与重置真实可用：查询按名称、单位、期间、类型、状态和负责人过滤；重置恢复默认。

### C. 状态统计

- 桌面宽屏一行 6 张：全部任务、草稿、待解析、生成中、待确认、已完成。
- 卡片约 `106px` 高，白底、细边框、`6px` 圆角、无重阴影；每张有状态色图标、数字、单位及“较昨日”辅助文案。
- 点击状态卡切换同义筛选；统计与列表同一数据源。

### D. 状态 Tab、表格和空态

- Tab：全部任务、执行中、待确认、已归档、异常任务；活动状态可视且可键盘操作。
- 桌面表头顺序：任务编号、任务名称、被审计单位、审计期间、任务类型、当前状态、更新时间、操作。
- 空态仍保留标题与表头；展示无任务插图、标题、说明、创建任务与导入模拟任务两个操作。
- 有数据态替换空态为数据行；任务详情、继续执行/处理、日志/归档按状态给出一个主要操作，其他信息不硬塞进窄列。

### E. 推荐演示路径

- 表格下方紧贴 `12px` 间距，约 `158px` 高。
- 五步均分：创建任务、选择能力、上传资料、生成结果、导出归档；有彩色圆形图标、步骤名、双行说明与连接箭头。
- 不用额外箭头列破坏等分；通过伪元素绘制连接箭头。

### F. 右栏

- 桌面右栏宽约 `228px`，顶端与统计区对齐；任务状态说明按：草稿、待解析、解析中、待生成、生成中、待确认、待复核、已完成、已归档排列。
- 每项含 `22px` 状态图标、标题和一句说明；底部保留浅蓝提示框。
- 有数据态在状态说明下方添加“今日待办与提醒”，不移除状态说明。

## 创建任务基础信息规格

- 顶部五步：基础信息、选择能力、资料选择、模板与输出设置、确认提交；初始必须停在“基础信息”。
- 基础信息可编辑字段：任务名称、被审计单位、审计期间（起止）、任务类型、负责人、参与人员、任务说明。
- 任务类型变化时，更新下一步的推荐能力、资料提示和默认模板提示。
- 保存草稿保存当前字段并给出状态反馈；下一步仅在必填字段完整时进入“选择能力”。
- 不能再把基础信息渲染成只读预填值，也不能将初始步骤置为“选择能力”。

## 响应式验收

| 视口 | 布局要求 |
| --- | --- |
| ≥1440 | 212px 侧栏；主区与 228px 右栏双列；6 个统计卡一行；筛选单行 |
| 1200–1439 | 侧栏约184px；统计 3×2；筛选 3 列；右栏仍独立；表格仅自身横滚 |
| 768–1199 | 侧栏收为图标/抽屉；主区单列；右栏移至表格后；统计 2×3；筛选 2 列 |
| <768 | 侧栏抽屉；筛选单列、创建按钮满宽、统计 2 列；任务以信息卡呈现；流程纵向或横滚 |

共同约束：页面根节点不得横向滚动；`min-width: 0` 必须作用于 Grid 子项；空态使用有上下限的 `min-height`，数据态随内容自然撑开，不用固定 `vh` 制造大面积空白。

## T7：图标与推荐路径精修（已确认方案）

**目标：** 消除 Unicode 字形混用造成的廉价感，使“推荐演示路径”的箭头稳定地落在相邻步骤图标的中轴线上。

### 设计决策

- 使用内联线性 SVG（`currentColor`）替代 `▤`、`♟`、`⇧` 等字符图标；不新增图标依赖，不改变状态色语义。
- 图标名称仅为内部映射：`tasks`、`draft`、`parse`、`generate`、`confirm`、`complete`、`archive`、`create`、`ability`、`upload`、`report`、`export`。
- 桌面端箭头由每个步骤的伪元素承担，锚定在本步骤图标圆心的右侧，使用细线和小箭头，而不是文字 `→`；不会覆盖标题或说明。
- 小于 `768px` 时流程纵向展示，隐藏水平连接箭头，保持现有清晰阅读顺序。

### 最小实施计划

1. 在 `src/views/tasks/TaskIcon.vue` 新建只负责 SVG 映射的展示组件；输入为 `name` 和可选 `size`，输出带 `aria-hidden` 的线性 SVG。
2. 在 `taskCenterData.js` 将各区域的 `icon` 字段换为语义名称；`TaskList.vue` 用 `TaskIcon` 渲染统计、流程、右栏图标。
3. 仅调整 `TaskList.vue` 内的路径图标尺寸、状态色背景与连接线定位；保留所有筛选、空态、表格和响应式断点。
4. 运行 `npm run check`，并在 `1584×996`、`390×844` 浏览器核验图标、箭头和根宽度。

- [x] T7.2：桌面箭头中轴定位与小屏隐藏
- [x] T7.3：定向构建与浏览器验收（`npm run check`、`npm run build` 通过；桌面/移动端无根横向溢出）

## T9：连接箭头几何中点修正

- 问题：T8 只固定了箭头的垂直中线，旧实现仍将其锚定在步骤卡右缘，文字宽度不同或卡片间隙变化时会偏离相邻图标的水平中点。
- 处理：桌面端将连接箭头的锚点设为“半个步骤列宽 +（图标直径与列间隙之和的一半）”；在当前 `56px` 图标、`26px` 列间隙下为 `41px` 偏移。
- 验收：每一个连接箭头的中心与相邻两个 `.flow-icon` 圆心的算术中点误差不超过 `1px`；移动端连接箭头仍隐藏。

- [x] T9.1：连接箭头水平锚点改为图标几何中点
- [x] T9.2：桌面与移动端几何验收（4 个连接点最大误差 `< 0.01px`；移动端箭头隐藏、根宽 `375 ≤ 390`）
