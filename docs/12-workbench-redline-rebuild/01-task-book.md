# 审计工作台红框复刻任务书

## 目标

严格对齐用户标注的审计工作台效果图，将 `/workbench` 改为两套状态：

- 空白页：新用户首次登录，指标均为 0，突出欢迎引导、推荐开始方式、任务流程预览和右侧新手引导。
- 有数据页：老用户工作台，突出待办、最近任务、任务进度、风险提醒、最近结果、快捷入口、系统通知和操作日志。

成功标准：页面第一屏结构、信息层级、模块命名、按钮入口和主要视觉密度与效果图一致；不保留原有“当前任务入口 / 固定任务入口 / 数据源健康”作为主结构。

## 二次密度修正目标

上一轮虽然覆盖了模块，但页面仍偏“卡片堆叠”。本轮必须按参考图压缩为密集后台：

- 侧栏宽度采用 `clamp(160px, 13.5vw, 220px)`，导航项高度约 40-44px。
- 顶栏高度约 58px，不显示效果图之外的演示切换控件。
- 指标卡高度约 80-88px。
- 有数据页第一业务行高度约 300px；第二业务行高度约 220px。
- 最近操作日志在 893px 高桌面视口内必须露出，目标起始位置不晚于 850px。
- 列表项使用紧凑行或表格样式，避免厚卡片堆叠。

## 三次自适应与细节修正目标

用户反馈仍有明显差异：大小尺寸不能只靠固定高度压缩，比例需要随视口自适应，图表和细节也要更像参考后台页面。

- 主内容不能依赖单一 1584x893 截图硬编码；数据页高度使用 `clamp()` 和 `calc()`，在 1366-1600px 宽桌面上保持协调。
- 数据页第一业务行目标高度：`clamp(280px, 32vh, 330px)`；第二业务行目标高度：`clamp(190px, 24vh, 240px)`。
- 指标区目标高度：`clamp(82px, 11vh, 104px)`；右侧创建按钮高度和指标区协调。
- 任务进度概览从纯柱状条升级为更接近参考图的“柱状 + 完成率折线”组合视觉。
- 图标不再只依赖单字块，使用统一线性符号/业务图形增强识别，但不引入新依赖。
- 表格和列表细节优化：文本不重叠，列宽比例稳定，按钮不挤压主体内容。
- 验证至少覆盖两个桌面视口：`1584x893` 和 `1366x768`；两者均应模块完整、无明显重叠，日志区在合理位置露出或可自然下滚查看。

## 当前实现差距

- 空白页已有指标、欢迎区、推荐入口和流程预览，但页面结构受 `PageHeader` 影响，顶部和效果图不一致。
- 有数据页当前是任务入口和卡片墙，不是效果图里的密集 dashboard。
- 右侧快捷入口、系统通知、风险提醒、最近生成结果、最近操作日志在当前有数据页缺失或不显性。
- 顶部指标卡缺少图标、较昨日变化、可点击箭头。
- 左侧导航和顶部栏的红白视觉与效果图存在差距，但全局改动应保持小范围。

## 区域拆解

### A. 全局外框

- 左侧导航：红色品牌头，白底菜单，当前“审计工作台”红色高亮，底部“收起导航”。
- 顶部栏：标题“审计工作台”，右侧通知、帮助、用户“审计管理员”。
- 主内容背景：浅灰白，所有模块以白色边框面板承载，边距紧凑。

### B. 顶部指标区

- 4 张指标卡：进行中任务、待确认异常、待复核报告、失败任务。
- 空白页数值均为 0。
- 有数据页数值分别使用 18、36、9、3，并展示“较昨日 +2 / +5 / +1 / -1”。
- 指标卡必须有图标、单位、跳转箭头或可点击暗示。
- 右侧保留主按钮“创建审计任务”。

### C. 空白页主体

- 欢迎区：标题“欢迎使用审计大模型系统”，红色提示“当前暂无审计任务”，说明文案，三个操作：创建审计任务、导入模拟数据、查看演示流程。
- 推荐开始方式：四张卡，分别是创建常规审计任务、从文件中心导入资料、体验知识库智能体、查看系统配置，每张卡包含图标、说明和小步骤。
- 任务流程预览：创建任务、上传资料、解析资料、生成分析、人工确认、保存版本、导出归档。
- 右侧栏：新手引导、系统初始化状态、最近操作空态。

### D. 有数据页主体

- 第一行：左侧“我的待办”，中间“最近审计任务”，右侧“快捷入口”。
- 我的待办：tab 包含全部、异常确认、报告复核、失败重试；列表至少 5 条；按钮包含“去处理”和“查看详情”。
- 最近审计任务：列包含任务名称、审计单位、审计期间、当前阶段、进度、状态、操作；进度用横向条展示。
- 快捷入口：2 列 3 行图标矩阵，创建任务、上传资料、知识库问答、制度比对、费用异常监控、报告生成。
- 第二行：任务进度概览、风险提醒、最近生成结果、系统通知。
- 任务进度概览：柱线组合视觉，展示进行中、待确认、已完成、失败、已归档。
- 风险提醒：TOP5 风险，含等级、单位、发现时间、查看依据。
- 最近生成结果：制度差异清单、费用异常汇总、审计报告草稿等，支持查看/下载。
- 系统通知：通知点颜色区分类型，显示时间。
- 底部：最近操作日志，列包含时间、操作内容、对象、操作人。

## 实施任务

### Task 3: 有数据页复刻

- 实现我的待办、最近审计任务、快捷入口。

- 待办、任务、风险、结果、通知、日志均有可扫读数据。

### Task 4: 外框视觉对齐

- 顶栏标题与右侧图标布局接近效果图。

文件：`scripts/check-demo-contract.js`

- 移除对旧结构的依赖。

## 每次继续工作前检查

- 先读本任务书。
- 确认当前要做的是 Task 1-6 的哪一项。
- 不做与工作台复刻无关的重构。
- 每次代码改动后只跑相关检查：`npm run check`、`npm run build`，以及工作台浏览器验证。
