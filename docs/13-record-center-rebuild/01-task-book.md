# 记录中心页面重构任务书

## 目标

只修改 `src/views/records/RecordCenter.vue`，复刻用户给出的记录中心空页面和有数据页面。页面顶部增加“空页面 / 有数据页面”切换按钮，用于同一地址对比两种状态。不得改动其他页面、路由、全局布局或 mock 数据。

## 业务口径

- 记录中心不是人工录入页面，而是系统自动沉淀关键操作和审计证据的地方。
- 空页面强调自动记录范围和追溯流程。
- 有数据页面强调筛选、分类 Tab、全过程留痕、查看详情、前后变化、关联任务、关联文件、关联版本、复核意见和导出文件。

## 区域拆解

### 1. 顶部统计区

- 五张卡片：版本记录、操作留痕、导出记录、复核记录、异常操作。
- 空态数值均为 `0 条`，趋势为 `较昨日 0`。
- 有数据态分别为 `128 条`、`3,426 条`、`286 条`、`74 条`、`12 条`。
- 有数据态趋势分别为 `较上月 +18 条 ↑`、`+326 条 ↑`、`+32 条 ↑`、`+9 条 ↑`、`-3 条 ↓`。
- 宽屏五列，中等宽度压缩，窄屏换为两列或一列。

### 2. 筛选 + Tab + 表格

- 筛选字段：所属任务、对象类型、对象名称、操作人、部门、操作类型、时间范围、操作结果。
- 操作按钮：查询、重置、导出日志。
- Tab：全部记录、版本记录、操作留痕、导出记录、复核记录；当前选中操作留痕。
- 有数据态 Tab 带数量：全部记录（3,926）、版本记录（128）、操作留痕（3,426）、导出记录（286）、复核记录（74）。
- 表头：操作时间、操作人、部门、操作类型、对象类型、对象名称、操作内容、操作结果、操作入口、操作。

### 3. 空态表格

- 居中展示浅灰插画、标题 `暂无操作留痕`。
- 说明文字必须包含：创建任务、上传文件、生成结果、确认异常、保存版本、提交复核和导出文件后，系统会自动在这里记录全过程。
- 主按钮：创建审计任务。
- 分页保留：共 0 条、当前页 1、10 条/页、跳至 1 页。

### 4. 有数据表格

- 行数据覆盖：创建任务、上传文件、解析文件、生成结果、确认异常、排除异常、修改报告、保存版本、提交复核、导出文件、配置规则、变更权限。
- 操作类型使用彩色标签，操作结果使用绿色 `成功` 标签。
- 操作列包含 `查看详情`、`下载凭证`，部分行包含 `查看前后变化`。
- 分页显示共 3,426 条，页码 1、2、3、4、5、...、343。

### 5. 底部追溯流程

- 仅空态展示。
- 节点：任务、文件、结果、报告、版本、导出。
- 说明：创建审计任务、上传与解析资料、模型生成结果、报告生成与编辑、保存版本记录、导出与归档管理。
- 宽屏一行，窄屏换行，节点之间使用虚线方向连接。

### 6. 右侧空态说明

- 标题：记录中心说明。
- 小标题：自动记录范围。
- 列表项：创建任务、上传文件、解析文件、生成结果、确认异常、排除异常、修改报告、保存版本、提交复核、配置规则、变更权限。
- 每项包含图标、标题、说明。

### 7. 有数据右侧详情

- 标题：记录详情，右上角关闭按钮。
- 基本信息：操作时间、操作人、部门、操作类型、对象类型、对象名称、操作结果、操作入口。
- 分组：操作前后变化、关联任务、关联文件、关联版本、复核意见、导出文件、操作流程追溯。
- 时间线：创建任务、上传资料、模型生成（解析文件）、人工确认、保存版本、提交复核、导出归档；当前节点红色高亮。
- 底部按钮：下载日志、查看任务详情。

## 响应式要求

- 桌面宽屏：主区域 + 右侧栏双列，右侧栏约 260-320px。
- 小于 1180px：右侧栏下移，主表内部横向滚动。
- 小于 980px：统计卡、筛选区、流程节点自动换行。
- 不允许大面积空白、文字重叠或按钮撑破容器。

## Agent 分工提示词

### 视觉拆解 agent

只基于用户给的两张记录中心截图和文字说明，输出细分区域拆解和验收清单，不修改文件。重点拆顶部统计卡、筛选区/Tab/表格空态、底部追溯流程、右侧空态说明、有数据态主表和右侧记录详情。

### 代码约束检查 agent

只读检查 Vue/Vite 项目中记录中心页面、全局布局和可用组件，回答只改 `src/views/records/RecordCenter.vue` 是否足够、哪些全局样式会影响复刻、响应式风险和验证命令。

### 实现 agent

只负责 `src/views/records/RecordCenter.vue`，根据本任务书实现空页面和有数据页面切换、顶部统计、筛选、Tab、表格、空态、追溯流程、右侧说明和详情栏。不要修改其他文件。

### 验收 agent

只读检查 `/records` 页面代码是否覆盖任务书全部区域，运行轻量验证命令，指出遗漏、响应式风险和明显视觉偏差。不要修改文件。

- `npm run dev -- --port 5174` 后访问 `http://127.0.0.1:5174/records`
