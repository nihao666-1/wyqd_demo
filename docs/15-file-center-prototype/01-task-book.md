# 文件中心页面重构任务书

## 目标

只修改 `src/views/files/FileCenter.vue`，不改动其他页面、路由和全局样式。文件中心必须同时提供“空白状态”和“有数据状态”，并在页面顶部提供切换按钮。页面要严格对齐两张效果图红框区域，表达文件中心是制度文件、监管案例、行业研究、监督共享文件、费用数据、业务数据、报告模板、生成报告和导出文件的统一资产库，而不是简单附件列表。

## 效果图区域拆解

### 空白状态

1. 顶部筛选和指标区
   - 标题为“文件中心”。
   - 筛选字段：文件名称、文件类型、所属单位、来源、上传人、解析状态、权限范围。
   - 操作按钮：查询、重置、上传文件。
   - 指标卡：文件总数、待解析、解析成功、已引用、导出文件，数值均为 0。
   - 布局要求：筛选区两行自适应，指标卡一行五列，宽度不足时自动换行。

2. 中部文件列表区
   - 分类 Tab：制度文件、监管案例、行业研究、监督共享文件、费用数据、业务数据、报告模板、生成报告、导出文件。
   - 表格保留表头：复选框、文件名称、文件类型、所属单位、来源、上传时间、解析状态、引用次数、权限范围、操作。
   - 空状态文案：文件库为空。
   - 空状态入口：上传文件、上传文件夹、从任务导入。
   - 分页保留“共 0 条、上一页、1、下一页、10 条/页、跳至 1 页”。

3. 右侧资产接入说明区
   - 文件接入流程：上传文件、解析资料、补全元数据、建立引用、加入任务。
   - 支持格式：PDF、Word、Excel、ZIP。
   - 说明：单个文件最大 500MB、单次最多 20 个文件、解析状态影响文件引用与分析结果。
   - 布局要求：右侧宽度约 260-300px，窄屏时下移并保持内容完整。

### 有数据状态

1. 顶部功能和筛选区
   - 功能 Tab：文件管理、文件查询、文件详情、引用关系、解析状态。
   - 筛选字段：文件名称、文件类型、所属单位、来源、上传人、上传时间、解析状态、权限范围。
   - 操作按钮：查询、重置。
   - 每个功能 Tab 点击后必须有相应内容反馈，不允许空白。

2. 文件列表区
   - 分类 Tab：全部文件、制度文件、监管案例、行业研究、监督共享文件、费用数据、业务数据、报告模板、生成报告、导出文件，并显示数量。
   - 表格字段：复选框、文件名称、文件类型、所属单位、来源、上传人、上传时间、解析状态、引用次数、权限范围、操作。
   - 操作按钮：查看、下载、加入任务、查看引用、作废文件。
   - 点击“查看引用”必须更新右侧引用详情。
   - 分页显示：共 2,356 条，页码 1-5、省略号、236、每页 10 条、跳至 1 页。

3. 右侧引用详情区
   - 文件基础信息：图标、文件名、类型、大小、上传时间、解析状态。
   - 主操作：加入任务、查看引用、作废。
   - 引用的任务：任务名、引用时间、引用次数。
   - 引用的报告章节：报告和章节名、引用时间。
   - 来源快照：来源路径、发布时间、最新更新。
   - 解析元数据：解析模型、解析时间、页数、条款数、关键词。
   - 操作历史：操作者、时间、动作、结果。

## 多 Agent 分工

### Agent A：现有实现定位

提示词：在工作区中定位“文件中心”页面相关文件、共享样式和脚本。只做代码阅读，不修改文件。输出页面入口、现有功能、明显缺口、谨慎避免影响的共享文件、建议最小修改范围。

### Agent B：空白状态拆解

提示词：只分析文件中心空白页效果图。细拆红框区域：顶部筛选和指标、中部分类和空表格、右侧流程/格式/说明。输出功能点、尺寸比例、间距、按钮交互、响应式风险。

### Agent C：有数据状态拆解

提示词：只分析文件中心有数据页效果图，重点是红框区域：顶部功能 Tab 和筛选区、分类 Tab 和表格分页、右侧文件引用详情面板。输出字段、功能点、按钮交互、子页面内容、尺寸比例、响应式风险。
