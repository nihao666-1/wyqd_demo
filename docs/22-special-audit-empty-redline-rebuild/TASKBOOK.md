# 专项审计分析空白页复刻任务书

## 目标边界

- 目标路由：`/regulatory/workbench`。
- 目标效果：复刻用户提供的“专项审计分析空白页”效果图，而不是继续沿用现有“监管分析工作台”。
- 数据状态：全部为空，顶部统计均为 0，最近任务表为空。
- 功能定位：承接非固定报表类专题分析，将监管案例、舆情动态、监督共享文件与数据标签转化为可确认、可引用、可进入报告的审计线索。
- 改动原则：优先最小闭环，页面级重构可以接受；不触碰无关模块行为，不清理历史改动。

## 红框区域拆解

### A. 顶部框架

- 左侧导航宽约 210px，品牌栏红底白字。
- 顶部栏高约 60px，左侧显示 `审计工作台 / 专项审计分析`，右侧为通知、帮助、分隔线、用户。
- 当前主导航为 `专项审计分析`，红色高亮，左侧 4px 红条。
- 当前项下方展开两个子项：`监管案例舆情分析`、`监管共享信息分析`，灰色圆点，缩进展示。

### B. 页面标题

- 主内容第一行是大标题 `专项审计分析`。
- 标题位于内容区左上，和顶部统计卡保持 16-18px 间距。

### C. 顶部统计红框

- 5 张横向统计卡，统一高度约 116px。
- 内容依次为：分析任务 0个、监管关注点 0条、舆情风险 0条、共享文件 0份、汇总报告 0份。
- 每张卡左侧为彩色图标块，右侧标题、粗体数字、`较昨日 -`。
- 卡片之间有 16-18px 间距，边框浅灰，圆角不超过 6px。

### D. 中部空状态红框

- 大面板高度约 400px，左侧为淡灰资料/放大镜插画，中间为主说明和两个能力入口。
- 主标题：`暂无专项审计分析任务`。
- 描述：`可从监管案例舆情或监督共享信息发起分析，形成监管关注点、审计检查建议和汇总分析报告。`
- 两个能力卡横向排列，尺寸一致，红框内宽约 320px，高约 240px。
- 卡 1：`监管案例舆情分析`，步骤为 `输入单位/业务 -> 获取案例舆情 -> 加入审计重点`，按钮 `新建案例舆情分析`。
- 卡 2：`监督共享信息分析`，步骤为 `选择共享文件 -> 提取标签 -> 生成汇总报告`，按钮 `新建共享信息分析`。
- 两个按钮为白底红边红字。

### E. 右侧资料说明红框

- 右侧上方面板标题：`所需资料说明`。
- 内含 4 张资料卡：`监管案例库`、`舆情动态`、`合规/风险/审计共享文件`、`数据标签`。
- 每张资料卡左侧彩色图标，右侧标题加两行说明；垂直间距约 10px。
- 面板宽约 260-280px，和主区有 14-18px 间距。

### F. 最近分析任务红框

- 位于中部面板下方，占左侧主列宽。
- 标题 `最近分析任务`。
- 表头：任务编号、分析类型、被分析单位、分析期间、状态、输出结果、操作。
- 空状态居中：灰色收纳盒图标、`暂无数据`、`当前暂无专项审计分析任务，快去创建吧！`。
- 右下角链接：`查看全部任务 >`。

### G. 使用提示红框

- 右侧下方面板标题：`使用提示`，灯泡橙色。
- 三条要点：
  - 可按组织合并类分析能力使用；
  - 分析结果可引用到审计报告；
  - 所有过程留痕，支持追溯与复核。

## 自适应要求

- 1600px 左右：对齐效果图，左主列 + 右栏同屏展示。
- 1366/1440px：仍保持左右两列，统计卡可等比压缩，避免主区大面积空白或横向溢出。
- 1024-1199px：右栏可落到下方，两张能力卡保持双列或按可用宽度收缩。
- 768px 以下：统计卡、能力卡、右侧资料说明改为单列，导航走现有窄屏规则。
- 不使用固定大宽度制造空白；关键区域使用 `minmax()`、`clamp()` 和自适应高度。
